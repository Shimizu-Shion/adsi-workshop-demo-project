# 打刻メモ入力 — 要件 Q&A

## 概要

打刻（出勤・退勤）時にメモを入力できる機能を追加する。

---

## 確定仕様まとめ

### 機能要件

| 項目 | 仕様 |
|------|------|
| 対象 | 出勤・退勤の両方 |
| 入力 | 任意（空欄OK） |
| 最大文字数 | 200文字 |
| 入力制限 | なし（改行・絵文字すべてOK） |
| 入力欄 | 複数行 textarea、常に表示 |
| 表示 | 出勤メモ欄・退勤メモ欄の2つを分けて表示 |
| 後編集 | 可能（期限なし） |
| 編集UI | 本日の打刻記録でメモをダブルクリック → インライン編集 |
| 編集キャンセル | Escキー |
| 月次履歴 | メモ列として表示（閲覧のみ、編集不可） |
| チーム一覧 | 非表示 |
| 検索 | 初回リリースに含めない |

### データ設計

| カラム | 型 | 備考 |
|--------|-----|------|
| `clock_in_memo` | `VARCHAR(200)` | 出勤メモ。NULL許容 |
| `clock_out_memo` | `VARCHAR(200)` | 退勤メモ。NULL許容 |

### 同日複数回出退勤（追加要件）

- 退勤後に再度出勤ボタンを押せるようにする（現在は退勤後ロック）
- 同日に複数レコードを記録可能（例: 午前勤務 + 午後勤務 = 2レコード）
- 再出勤時は新レコードの出勤メモ欄が表示される

### 想定ユースケース

勤務形態（在宅/本社/客先）、本日の予定、週次予定など自由記述。

## メモの基本仕様

[Question] メモは出勤・退勤の両方で入力可能にしますか？それとも片方のみ？
[Answer]両方可能

[Question] メモの最大文字数は何文字にしますか？（例: 100文字、200文字、500文字）
[Answer]200文字

[Question] メモの入力は任意（空欄OK）ですか？それとも必須にしますか？
[Answer]任意

[Question] メモに入力できる内容に制限はありますか？（例: テキストのみ、改行あり/なし、絵文字OK/NG）
[Answer]制限なしで問題がなければ制限なし　問題があれば伝えて

## 追加質問（回答から生じた疑問）

[Question] メモの後編集に期限はありますか？（例: 当日中のみ / 翌日まで / 無期限）
[Answer]無制限

[Question] メモ編集はどこから行いますか？（例: 本日の打刻記録の「編集」ボタン / 月次履歴からも編集可能）
[Answer]メモをダブルクリックすると該当メモが編集可能

[Question] 検索機能は初回リリースに含めますか？それとも将来対応（今はカラムだけ用意しておく）？
[Answer]含めない

[Question] 出勤メモと退勤メモで入力欄を分けて2つ表示しますか？（出勤時は出勤メモ欄のみ表示、退勤時は退勤メモ欄のみ表示、という切り替え方式？）
[Answer]二つ用意　ただ、退勤後出勤ボタンを押せるように変更してほしい　今は退勤後出勤ができない
